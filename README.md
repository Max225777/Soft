# Менеджер ниш и статистики Lolzteam Market

Настольное приложение для автоматизации управления продажами, мониторинга и аналитики аккаунтов на маркетплейсе **Lolzteam Market**. Реализует группировку аккаунтов по настраиваемым нишам, массовые действия (поднятия, закрепления, изменение цен), отслеживание продаж и расчёт чистой прибыли.

## Быстрый старт

Для опытных пользователей (Linux / macOS):

```bash
git clone https://github.com/Max225777/Soft.git
cd Soft
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # затем откройте .env и впишите ваш LOLZ_API_TOKEN
python -m app
```

Для Windows (PowerShell):

```powershell
git clone https://github.com/Max225777/Soft.git
cd Soft
py -3.11 -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
Copy-Item .env.example .env
# откройте .env в редакторе и впишите ваш LOLZ_API_TOKEN
python -m app
```

## Установка (пошагово)

### 1. Клонирование репозитория

```bash
git clone https://github.com/Max225777/Soft.git
cd Soft
```

### 2. Создание виртуального окружения

Виртуальное окружение изолирует зависимости приложения от системного Python.

**Linux / macOS:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows (PowerShell):**
```powershell
py -3.11 -m venv .venv
.venv\Scripts\Activate.ps1
```

> Если PowerShell блокирует выполнение скрипта активации, разрешите это командой:
> `Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned`

**Windows (cmd):**
```cmd
py -3.11 -m venv .venv
.venv\Scripts\activate.bat
```

После активации в начале строки появится префикс `(.venv)`.

### 3. Установка зависимостей

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Установка может занять несколько минут — особенно пакет `PySide6` (~100 МБ).

### 4. Настройка API-токена Lolzteam Market

1. Откройте [https://lzt.market/account/api](https://lzt.market/account/api) и войдите в свой аккаунт.
2. Создайте новый API-токен со scope **`read+post+market`**.
3. Скопируйте полученный токен.
4. В корне проекта скопируйте файл `.env.example` в `.env`:
   ```bash
   cp .env.example .env          # Linux / macOS
   Copy-Item .env.example .env   # Windows PowerShell
   ```
5. Откройте `.env` в любом редакторе и впишите токен:
   ```dotenv
   LOLZ_API_TOKEN=ваш_токен_сюда
   LOLZ_API_BASE_URL=https://prod-api.lzt.market/
   LOLZ_API_LANG=ru
   ```

> **Безопасность.** Файл `.env` добавлен в `.gitignore` и никогда не попадёт в репозиторий. Токен также можно ввести внутри приложения (вкладка «Настройки» → «API»), где он сохраняется в зашифрованном виде в локальной БД.

### 5. Инициализация базы данных

При первом запуске приложение само создаст SQLite-файл `data/app.db` и применит схему. Если нужно сделать это заранее:

```bash
python -m app.db.init
```

Для ручного применения миграций (когда они появятся):

```bash
python -m app.db.migrate
```

## Запуск приложения

Убедитесь, что виртуальное окружение активировано, и выполните:

```bash
python -m app
```

Альтернативный способ — напрямую через точку входа:

```bash
python app/main.py
```

После запуска откроется главное окно с двумя вкладками: **«Главная»** (ниши и аккаунты) и **«Статистика»**. При первом запуске без токена приложение сразу предложит открыть раздел настроек.

### Полезные флаги командной строки

| Флаг | Описание |
|:---|:---|
| `--debug` | Запуск с подробным логированием (уровень DEBUG). |
| `--no-auto-cycle` | Не запускать 20-минутный цикл автоматически при старте. |
| `--config <path>` | Использовать альтернативный файл настроек. |
| `--db <path>` | Использовать альтернативный файл БД. |

Пример:
```bash
python -m app --debug --no-auto-cycle
```

## Часто задаваемые вопросы

**Как получить API-токен?**
Зайдите на [https://lzt.market/account/api](https://lzt.market/account/api), создайте токен со scope `read+post+market` и вставьте его в `.env` или в настройки приложения.

**Где хранится база данных?**
По умолчанию — в `./data/app.db` рядом с приложением. Путь можно изменить флагом `--db` или в настройках.

**Можно ли использовать несколько аккаунтов Lolzteam?**
В текущей версии — один аккаунт (один токен). Поддержка нескольких токенов заложена в Roadmap (Фаза 3).

**Почему приложение делает паузы между запросами?**
API Lolzteam Market ограничивает частоту: 120 запросов/мин для обычных эндпоинтов и 20 запросов/мин для поисковых, с минимальной задержкой 3 секунды между запросами. Менеджер очереди соблюдает эти ограничения автоматически.

**Как часто обновляются данные?**
Автоматический цикл запускается каждые **20 минут** (интервал настраивается). Вручную можно обновить в любой момент кнопкой «Обновить» на вкладке «Главная».

## Решение проблем

**`ModuleNotFoundError: No module named 'PySide6'`**
Не активировано виртуальное окружение или не установлены зависимости. Выполните `source .venv/bin/activate` (или `.venv\Scripts\Activate.ps1`) и `pip install -r requirements.txt`.

**`401 Unauthorized` при запросах к API**
Неверный или просроченный токен. Сгенерируйте новый на [https://lzt.market/account/api](https://lzt.market/account/api) и обновите `.env` / настройки.

**`429 Too Many Requests`**
Превышен лимит API. Приложение автоматически увеличит задержку и повторит запрос. Если ошибка повторяется — увеличьте в настройках `Минимальную задержку между запросами` (по умолчанию 3 сек).

**Приложение не запускается на Linux: `qt.qpa.plugin: Could not load the Qt platform plugin "xcb"`**
Установите недостающие системные библиотеки:
```bash
sudo apt install libxcb-cursor0 libxkbcommon-x11-0 libxcb-icccm4 libxcb-image0 libxcb-keysyms1 libxcb-randr0 libxcb-render-util0 libxcb-shape0 libxcb-xinerama0 libxcb-xkb1
```

**Ошибка при сборке SQLCipher (`pysqlcipher3`)**
На Ubuntu/Debian установите dev-пакеты:
```bash
sudo apt install libsqlcipher-dev build-essential python3-dev
```
На Windows проще использовать обычный SQLite без шифрования БД (шифрование токена при этом сохраняется — он шифруется AES-256 отдельно).

**Логи**
Все события пишутся в `logs/app.log`. Уровень логирования настраивается в разделе настроек или флагом `--debug`.
